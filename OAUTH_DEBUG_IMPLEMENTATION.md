# Google OAuth 404 Debugging - Implementation Summary

## Changes Made

### 1. Mobile App Console Logging (`mobile/services/api.ts`)

Added comprehensive diagnostic logging that displays when the user initiates Google login:

```
[Google OAuth] Configuration:
  - Client ID: <your-client-id>.apps.googleusercontent.com
  - Reversed Client ID: com.googleusercontent.apps.<id>
  - Redirect URI: com.googleusercontent.apps.<id>://oauthredirect
```

This appears in the **Metro console** (where you see Expo debug messages) before Google's login screen opens.

**What it helps detect:**
- The exact redirect URI your app is using
- Whether client IDs are correctly configured in app.json

### 2. Enhanced Error Logging (`mobile/services/api.ts`)

When OAuth fails, logs detailed error information to help diagnose issues:
- Auth result type
- Full response object
- Instructions on what to check in Google Cloud Console

### 3. Improved Error Messages in UI (`mobile/app/login.tsx`)

When a 404 error occurs, the alert now suggests checking the console logs and provides guidance.

## Next Steps for User

### To Test This Fix:

1. **Clear app data**:
   ```bash
   # For iOS simulator
   xcrun simctl erase all

   # For Android emulator
   adb shell pm clear com.dukiai.app
   ```

2. **Start the mobile app**:
   ```bash
   cd mobile
   npm start
   ```

3. **Try Google login** on the iOS simulator:
   - Press "Continue with Google" button
   - Watch the **Metro console** for the `[Google OAuth] Configuration:` log output
   - Copy the exact `Redirect URI` value

4. **Check Google Cloud Console**:
   - Go to https://console.cloud.google.com
   - APIs & Services → Credentials
   - Find the **iOS** OAuth credential (important: must be iOS, not Web)
   - Edit it and verify the redirect URI from step 3 is listed under "Authorized redirect URIs"
   - If missing, add it and save

5. **Try again**: Clear app cache and attempt login again

### If Still Getting 404:

**Check that you're using an iOS app credential:**
1. In Google Cloud Console credentials list
2. Look for credential type labeled **"iOS"** (not "Web" or "Desktop")
3. If only a Web credential exists, you may need to create an iOS credential
4. The iOS credential should have the reversed client ID scheme configured

**For Expo Go Development:**
- The redirect URI will be: `exp://<local-ip>:<port>/--/auth/callback`
- This ALSO needs to be registered in Google Cloud Console if testing with Expo Go

## Technical Details

**Why the 404 happens:**
- Google OAuth requires the redirect URI to match exactly what's registered in Google Cloud Console
- For iOS native apps, the redirect URI must use a reversed client ID as the URL scheme
- If the scheme doesn't match registered URIs, Google returns a 404

**Why console logging helps:**
- The redirect URI is dynamically generated by `AuthSession.makeRedirectUri()`
- It depends on correct app.json configuration and AuthSession implementation
- Logging shows exactly what the app is sending to Google, making it easy to verify it's registered
